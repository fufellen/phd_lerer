# phd_lerer

Репозиторий содержит исследовательский численный код для электромагнитных
расчетов слоя/решетки.

Основная научная часть находится в папке `Полиномы 7_1`. Это C-программа для
расчета коэффициентов отражения `R`, прохождения `T` и поглощения
`P = 1 - R - T` для электромагнитной задачи со слоем/решеткой. Код считает
обе поляризации (`p` и `s`), сравнивает аналитические формулы с численным
решением методом коллокации и использует функции Грина, функции Бесселя,
полиномы Лежандра и вспомогательные матричные процедуры.

## Структура

- `Полиномы 7_1/` - основной расчетный C-код.
  - `VIBR.c` - точка входа: спрашивает число вариантов, имя входного файла и
    имя файла результатов.
  - `Vvod.c` - читает параметры варианта из входного файла.
  - `F_df_2.c`, `E_ext.c` - аналитические решения для слоя в вакууме и слоя на
    подложке.
  - `F_df_IE_MK.c` - численное решение СЛАУ методом коллокации.
  - `Fun mod Bessel.c`, `Fun.c`, `V_.c`, `FGrin_dop.c` и другие файлы -
    специальные функции и вспомогательные расчеты.
  - `A.c` - пример входных данных, несмотря на расширение `.c`.
- `newLIb/` - старая C-библиотека численных процедур: функции Бесселя и
  Макдональда, полиномы Лежандра/Эрмита/Гегенбауэра, квадратуры Гаусса,
  детерминанты, операции с комплексными числами, выделение матриц и
  итерационные решатели.
- `edp/` - воспроизводимые ЭДП/EIM и COMSOL-diagnostic расчеты для
  плазмонного Ag-волновода из разбора Лерера.
  - `Ag_.c` - исходные оптические константы серебра.
  - `metal_strip_w800/` - Python-скрипты ЭДП, Java-скрипты COMSOL Mode
    Analysis, маленькие CSV-экспорты COMSOL и итоговые CSV/PNG-графики для
    Ag-полоски `W=800 nm`.
  - Большие `.mph`, `.class`, `.log`, `.status` и `.recovery` файлы не
    хранятся в Git.
- `composite_ema/` - Python-порт `COMPOSIT.c` (эффективная eps диэлектрика с
  плазмонными наночастицами, формула Клаузиуса-Моссотти / Maxwell-Garnett)
  из статьи Лерера про поглощающие метаповерхности с ПНЧ. Материальная база
  Ag/Au/Cu/HfN/ZrN/TiN, самопроверка, воспроизведение `eps_eff(lambda)`.
  Подробности и границы применимости: `composite_ema/README.md`.
- `grating_2d_rcwa/` - независимый (не порт кода Лерера) Fourier Modal
  Method/RCWA солвер для расчёта `R, T, P` той же 2D-периодической решётки.
  Реализованы и провалидированы: TMM (против Френеля/Эйри), 1D RCWA (TE)
  и полновекторный 2D crossed-grating солвер цилиндров; воспроизведены
  рис. 2 (Au vs Cu) и рис. 5 (HfN vs TiN) статьи на уровне качественных
  критериев. Цель задачи и статус: `grating_2d_rcwa/README.md`.
- `vibr_t/` - оригинальная программа Лерера **VIBR_T** (метаповерхности с
  плазмонными наночастицами, метод объёмного ИДУ 2012 г.), полученная
  2026-07-04 через Льва Яковлева, и Python-порт её материальной библиотеки
  и композитных формул (`COMPOSITE`/`COMPOSITE_3`). Таблицы констант
  извлекаются из C автоматически. Зафиксированы два существенных дефекта
  оригинала: баг мнимой части в `COMPOSITE_3` и отсутствие УФ-данных Cu
  (экстраполяция ниже 517 нм); пересчёт рис. 2 на константах Лерера
  закрыл УФ-расхождение Au/Cu. Подробности: `vibr_t/README.md`.
- `nanowaveguides/` - C-код поиска постоянной распространения мод планарных
  плазмонных/диэлектрических волноводов (`N_W_Plan_real/`, `N_W_EDP/`) -
  отдельная от `Полиномы 7_1` задача (волновод, а не дифракция на решётке).
  Подробности, происхождение и список сознательно не перенесённых
  IDE/build-артефактов и Origin-графиков: `nanowaveguides/README.md`.

Подробные команды для ЭДП/COMSOL блока: `edp/metal_strip_w800/README.md`.

## Входные данные основного расчета

`VIBR.c` сначала запрашивает:

1. количество вариантов;
2. имя входного файла;
3. имя файла, куда дописывать результаты.

Каждый вариант во входном файле читается функцией `VVOD()` из `Vvod.c`.
Комментарии во входном файле обрамляются символом `/`; функция `fSkipComment`
из `newLIb/FSKCOM.C` пропускает такие комментарии перед чтением очередного
числа.

Порядок параметров одного варианта:

1. `TEST` - режим расчета: `1` для слоя в вакууме, `2` для слоя на подложке.
2. `TET` - угол падения в градусах.
3. `d h l` - период, толщина и ширина пластины.
4. `AF[1] AF[2] AF[3]` - диапазон частот: начало, шаг, конец.
5. `EPS[1] EPS[2]` - комплексная диэлектрическая проницаемость неоднородности.
6. `MU[1] MU[2]` - комплексная магнитная проницаемость неоднородности.
7. `h2` - толщина слоя/подложки.
8. `eps[2] mu__[2]` и `eps[3] mu__[3]` - параметры слоя и подложки.
9. `AM_X[1] AM_X[2] AM_X[3]` - диапазон числа базисных функций по `x`.
10. `AM_Y[1] AM_Y[2] AM_Y[3]` - диапазон числа базисных функций по `y`.
11. `AN_g[1] AN_g[2] AN_g[3]` - диапазон числа членов функции Грина.
12. `PRINT` - `0` не писать подробный вывод в файл, `1` писать.

Примеры наборов параметров лежат в `Полиномы 7_1/A.c`.

## Сборка и запуск

В корне добавлен `CMakeLists.txt` для сборки основной программы
`phd_lerer.exe`. Проект рассчитан на Windows/MSVC, потому что исходники
используют `conio.h`, `kbhit()`, `getch()` и старые MSVC-совместимые
заголовки библиотеки.

Требования:

1. CMake 3.16 или новее.
2. Visual Studio Build Tools или Visual Studio с компонентом C/C++.

Сборка через bat-файл:

```bat
build.bat
```

Сборка Release:

```bat
build.bat Release
```

Или вручную через CMake:

```powershell
cmake -S . -B build
cmake --build build --config Debug
```

После сборки исполняемый файл будет находиться в:

```text
build\bin\Debug\phd_lerer.exe
```

Для запуска программа в консоли попросит количество вариантов, имя входного
файла и имя файла результатов. Пример входных данных можно взять из
`Полиномы 7_1/A.c`.

## Отладка в VS Code

Да, проект можно отлаживать в Visual Studio Code. Для этого добавлены:

- `.vscode/tasks.json` - задача сборки `build Debug`;
- `.vscode/launch.json` - конфигурация `Debug phd_lerer (MSVC)`.

Нужно установить расширение Microsoft C/C++ (`ms-vscode.cpptools`), CMake и
MSVC Build Tools. После этого откройте папку проекта в VS Code, поставьте
breakpoint и запустите конфигурацию `Debug phd_lerer (MSVC)` из вкладки
Run and Debug. Перед стартом отладки VS Code выполнит `build.bat Debug`.

## Текущее состояние

- Расчетный C-код выглядит как архив рабочей исследовательской программы:
  логика и численные методы сохранены, сборочная инфраструктура восстановлена
  через CMake для активного расчетного пути.
- В репозитории есть старые служебные файлы Visual Studio и документы с
  заметками по численным функциям.
- Добавлен воспроизводимый Python/COMSOL блок `edp/metal_strip_w800` для
  проверки ЭДП-подобной модели Ag-полоски на подложке. Python-скрипты
  используют `Ag_.c`, строят аналитические таблицы/графики и постобрабатывают
  COMSOL CSV-экспорты с выбором ветви по полевым диагностическим признакам.
- Добавлен `composite_ema/` - Python-порт `COMPOSIT.c` для метаповерхностей
  с плазмонными наночастицами. Воспроизводит только формулу (1) статьи
  (эффективная eps композита), не полный расчет `R, T, P` решётки - солвер
  для полной задачи не найден среди доступных материалов. Статус:
  diagnostic, подробности в `composite_ema/README.md`.
- Добавлен `nanowaveguides/` - C-исходники двух вариантов солвера постоянной
  распространения планарных плазмонных волноводов (`N_W_Plan_real`,
  `N_W_EDP`), перенесённые из Obsidian и перекодированные в UTF-8. Это
  архив исходного кода: сборка/CMake для них не настроены (в отличие от
  `Полиномы 7_1`). `nanowaveguides/README.md` фиксирует, чем `N_W_EDP`
  отличается от `N_W_Plan_real`, и какие файлы (IDE-артефакты,
  Origin `.OPJ`, `.docx`-инструкции) сознательно оставлены только в Obsidian.
