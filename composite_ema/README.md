# Composite EMA (Клаузиус-Моссотти / Maxwell-Garnett) для метаповерхностей Лерера

Python-порт `COMPOSIT.c` - файла, приложенного к статье Лерера А.М. и др.
"Электродинамический анализ оптических поглощающих метаповерхностей,
содержащих диэлектрические слои с плазмонными наночастицами" (Obsidian:
`PhD/Scientific_study/Scientific_study/paper_analysis/from_Lerer/от Лерера/
Лерер АМ_1.md`).

## Что это и что не это

`COMPOSIT.c` считает только эффективную диэлектрическую проницаемость
`eps_eff(lambda)` диэлектрического слоя с плазмонными наночастицами -
формула (1) статьи (Клаузиус-Моссотти / Maxwell-Garnett). Это входные
данные для полной задачи дифракции на решётке, но не сама задача.

Полные коэффициенты отражения/прохождения/поглощения `R, T, P` (рис. 2-5
статьи) считает другая программа - решение объёмного интегрального
уравнения для двухпериодической решётки (см. ссылку [1] в статье:
Лерер А.М., РЭ, 2012, т.57, №11). Эта программа не найдена среди
материалов, доступных при портировании (см. "Источники и провенанс" ниже),
и здесь не воспроизводится. Соответственно, этот пакет **не** воспроизводит
рисунки 2-5 статьи напрямую - только физику формулы (1), из которой они
складываются.

## Источники и провенанс

- `source_c/COMPOSIT.c` - перекодированная в UTF-8 копия оригинала
  (оригинал был в Windows-1251, лежит в Obsidian рядом со статьёй как
  `attachments/COMPOSIT — копия.c`). Логика не менялась.
- `source_c/EPS_MET_AuAgCu.c` - материальная база Ag/Au/Cu, скопирована из
  **соседнего, но другого** проекта Лерера про плазмонные нановолноводы
  (`PhD/Scientific_study/Scientific_study/paper_analysis/from_Lerer/
  нановолноводы от Лерера/N_W_Plan - real/EPS_MET.c`). Это предположение:
  `libmat()`, которую вызывает `COMPOSIT.c`, не найдена ни в одном из
  доступных файлов; `EPS_MET_AuAgCu.c` - единственная найденная база с
  таким же стилем (локальная интерполяция Лагранжа по табличным `lambda,
  eps'-i*eps''` для Ag/Au/Cu). Материалы HfN и TiN, которые тоже
  упоминаются в статье (рис. 5), в доступных исходниках не найдены и не
  восстановлены.
- Комментарий в `COMPOSIT.c` даёт индексацию `n_mat`: `1-Cu, 2-Ag, 4-Au и
  т.д.` - индекс `3` и материалы за пределами Ag/Au/Cu не расшифрованы.

## Известные отличия Python-порта от буквального C

1. **Off-by-one граница окна интерполяции.** В оригинальных `Ag()/Au()/Cu()`
   переменная `q`/`qq` (ёмкость массива) на 1-2 больше реального числа
   заполненных точек. При приближении к самому верхнему краю таблицы это
   могло читать неинициализированный `double` из C-стека. Порт использует
   `len(xi) - 1` (реальное число точек) - в пределах рабочего диапазона
   этого пакета (300-2000 нм) результат идентичен оригиналу.
2. **"Осиротевшая" точка Cu.** В оригинале `fss[84]` (пара `eps` при
   ~4000 нм) задан без соответствующего `xii[84]` (массив длин волн
   заполнен только до индекса 83, т.е. 3900 нм). Это дефект исходных
   данных, а не порта. Точка отброшена.
3. `divide_()`/`mult_()` (комплексное деление/умножение, тела не найдены)
   заменены на арифметику `complex` в Python - математически то же самое.

## Структура

- `source_c/` - перекодированные копии исходного C (см. выше).
- `scripts/composite_ema.py` - порт `COMPOSITE()`/`COMPOSITE_3()` +
  материальная база Ag/Au/Cu + самопроверка + скрипт воспроизведения.
- `results/composite_eps_Au_vs_Cu.csv` - `eps_Au`, `eps_Cu`,
  `eps_eff` (`C=10%`, `n_host=1.77`, как в статье) по сетке длин волн.
- `results/composite_eps_Au_vs_Cu.png` - график `Re/Im(eps_eff)(lambda)`
  для Au и Cu с отмеченным условием Фрёлиха.

## Запуск

```powershell
python .\composite_ema\scripts\composite_ema.py
```

Требует `numpy` и `matplotlib` (без `scipy`).

## Самопроверка

Скрипт при каждом запуске сверяет:
- `composite_from_eps()` с ручной аналитической формулой Maxwell-Garnett на
  синтетических (не табличных) `eps` при нескольких `C`, включая `C=0`
  (должно точно вернуть `eps_h`);
- `eps_au(530 nm)` и `eps_cu(517 nm)` с точными табличными узлами.

Обе проверки проходят (`assert`, аварийный останов при расхождении).

## Результат (текущий прогон)

Параметры взяты из статьи: слой с ПНЧ, `n_host=1.77` (`eps_h=3.1329`),
концентрация `C=10%`.

- Условие Фрёлиха `Re[eps_NP] + 2*eps_h = 0`: **Au ~540 нм**, **Cu ~528 нм**.
- Экстремум `|Im(eps_eff)|` (пик резонансных потерь): и Au, и Cu - около
  **554 нм** (сетка 2 нм), но у Au пик почти в 3 раза глубже (`Im≈-5.8`
  против `Im≈-1.9` у Cu) - совпадает с известной картиной: межзонное
  поглощение Cu ближе к её плазмонному резонансу сильнее гасит и уширяет
  пик, чем у Au.
- Резонанс на 530-560 нм согласуется с утверждением статьи ("Заключение"):
  "в этом диапазоне [500-600 нм] находится частота локализованного
  плазмона у золотых наночастиц".
- **Не проверено**: сравнение Au/Cu в УФ (<400 нм) из рис. 2 статьи. Табличные
  данные для Cu в этом файле начинаются только с 517 нм - в найденных
  исходниках УФ-данных для Cu нет, поэтому прямое сравнение с рис. 2
  недоступно на этом материале.

Статус: **diagnostic** - формула (1) статьи и материальная база
воспроизведены и самосогласованы (самопроверка проходит, резонанс
физически на своём месте), но полный результат статьи (`R, T, P` рис. 2-5)
этим пакетом не проверялся и не воспроизводился - для этого нужен другой,
не найденный солвер.
